# Organizing Folders in Obsidian: Explain Like I'm 5

## The Big Question
You just got a big empty toy box (Obsidian vault). How do you sort your toys so you can always find the one you want? Nick Milo walks through the journey from zero notes to 7,000+, showing that the best folder system is one you build yourself from the ground up -- not one you copy from the internet.

---

## Start With Nothing
When you have 2 notes, you don't need folders. That'd be like buying a filing cabinet for two sticky notes. Only add folders when the *pain* of not having them is real. Nick demos vaults at 0, 2, 22, and 75 notes to show when folders start earning their keep.

---

## The Fuzzy Folder Problem
Some notes are easy to file -- projects go in "Projects", daily notes go in "Calendar." But *idea* notes are tricky. Is "Yin and Yang" an atomic note? A knowledge note? A thought? Nick's key insight: **if the stuff is fuzzy, the folder name should be fuzzy too.** That's why he uses a catch-all called "Dots" for all idea/knowledge notes.

---

## Two Popular Frameworks

| | PARA (Tiago Forte) | ACE (Nick Milo) |
|---|---|---|
| **Letters** | Projects, Areas, Resources, Archives | Atlas, Calendar, Efforts |
| **Strengths** | Clear action orientation | Covers time-based + idea notes naturally |
| **Weakness** | No obvious home for daily notes / journal | Atlas can get large without sub-folders |
| **Best for** | Productivity-focused users | Thinkers who also manage projects |

Both work. Neither is "right." They're starting points you customize.

---

## Nick's Personal Setup (7,000+ notes)
- **Atlas** -- split into Dots (ideas), Maps (index notes), Works (drafts being shared)
- **Sources** -- pulled out of Atlas for incoming highlights/references
- **Calendar** -- daily notes + a "Compass" sub-folder for deep life reflections
- **Efforts** -- four intensity levels: On, Ongoing, Simmering, Sleeping
- **Extras (X)** -- images, PDFs, templates, scripts

He built this over *years*. He explicitly says: **don't copy this.** Build from the bottom up.

---

## The Bottom-Up Principle
1. Start with zero folders
2. Add notes until you feel friction finding things
3. Create a folder only when the pain justifies it
4. Revisit every few months -- your life changes, your folders should too

---

## Close the Sidebar
Nick's parting reminder: every minute organizing folders is a minute *not* spent linking ideas, making connections, and doing the actual thinking. Set up folders just well enough, then close the sidebar and get to work.

---

## The One-Sentence Version
**Don't copy anyone's folder system -- start with none, add folders only when finding notes becomes painful, and let your structure grow from your actual needs.**
